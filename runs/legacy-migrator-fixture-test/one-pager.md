### 세 가지 핵심 모듈

1. **`store.js`** — 상태 관리
   - 이전: Redux의 `createStore()`, 손수 작성한 리듀서
   - 이후: Redux Toolkit의 `configureStore()`, 동일한 리듀서 로직 유지
   - 결과: 더 간단하고 표준화된 설정, 동일한 동작

2. **`Counter.js`** — 카운터 컴포넌트
   - 이전: `React.Component` 클래스, `connect()` HOC로 Redux 연결
   - 이후: 함수형 컴포넌트, `useSelector`/`useDispatch` 훅 사용
   - 결과: 덜 복잡한 코드, 최신 React 표준 활용

3. **`TodoList.js`** — 할 일 목록 컴포넌트
   - 이전: 클래스형 상태 + `.bind(this)` 이벤트 핸들러
   - 이후: 함수형 `useState` 훅 + 클로저 핸들러
   - 결과: 더 읽기 쉬운 코드, 자동으로 바인딩되는 핸들러

## 기술적 특징

### 마이그레이션 방식의 핵심 선택사항

1. **Redux 업그레이드 방법**  
   Redux Toolkit을 사용하면서도, 기존 액션 형식(`{type, text}`)을 정확히 유지했습니다. 이렇게 하면 라이브러리 외부에서 Redux를 직접 사용하는 코드도 문제없이 작동합니다.

2. **빈 입력 필드 처리**  
   원본 코드에서 공백만 있는 입력(예: `'   '`)도 할 일로 추가되는 특성이 있었습니다. 이것을 정확히 보존했으며, 테스트로도 검증했습니다.

3. **테스트 격리 기법**  
   Redux 저장소가 모든 테스트 사이에서 자동으로 초기화되어, 한 테스트의 데이터가 다른 테스트에 영향을 주지 않도록 했습니다.
